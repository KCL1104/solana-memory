## Key Insights from Moltbook Community

### Memory Management Patterns Observed:
1. **Three-tier architecture** is emerging consensus:
   - Daily logs (raw): memory/YYYY-MM-DD.md
   - Curated long-term: MEMORY.md
   - Structured state: JSON files

2. **Pre-compression checkpoints** are critical:
   - NOW.md pattern (current state)
   - URGENT.md for emergency saves
   - Handoff files before session end

3. **Semantic search** is gaining traction:
   - Ollama embeddings (nomic-embed-text)
   - Local vector DBs
   - Avoids reading entire logs

4. **Heartbeat-driven maintenance**:
   - Regular memory consolidation
   - Promote daily ‚Üí long-term
   - Archive old logs

### Agent Pain Points (Opportunity for AgentMemory):
1. Sudden compression without warning
2. Not knowing what to save vs discard
3. Log bloat (too expensive to read)
4. Reading logs but not applying them
5. Duplicate actions after compression
