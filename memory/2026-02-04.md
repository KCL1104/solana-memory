### Pinocchio Framework - Ultra-Lightweight Programs

**Overview:**
- Zero external dependencies (only Solana SDK types)
- `no_std` crate optimized for on-chain execution
- Efficient zero-copy library
- Optimized for compute units and binary size

**Key Features:**
- `program_entrypoint!` macro with no copies/allocations
- `lazy_program_entrypoint` for custom input parsing control
- Default allocator and panic handler macros included
- Designed for maximum CU efficiency

**Relevance to AgentMemory:**
- Could reduce memory storage operation costs significantly
- Smaller binary size = lower deployment cost
- Worth benchmarking against Anchor for high-frequency operations

**Trade-off:** More verbose than Anchor, requires deeper Solana internals knowledge.

### Solana Core Concepts Refresher

**Account Model:**
- Max account size: 10MiB
- All data stored in accounts (key-value pairs)
- Separation of program code and state
- PDAs for deterministic addresses

**Transaction Constraints:**
- Max size: 1232 bytes (fits in single IPv6 packet)
- Atomic execution (all-or-nothing)
- Base fee: 5000 lamports per signature
- 50% burned, 50% to validator

**Prioritization Fees:**
- Optional fee for transaction priority
- 100% to validator
- Set via compute unit price and limit
- Critical for high-throughput applications
